1.反转链表

迭代法

空间复杂度分析：遍历时只有3个额外变量，所以额外的空间复杂度是 O(1)。
时间复杂度分析：只遍历一次链表，时间复杂度是 O(n)。

递归法

编写递归代码首先要考虑清楚递归的函数（reverseList） 能做什么，它可以反转链表并返回新链表的头结点，即原链表的尾结点。所以我们先递归处理 reverseList(head -> next)，将以head -> next为头结点的链表反转，并得到这段原链表的尾结点tail，此时head -> next是这段新链表的尾结点，令它的next指针指向head，并将head -> next指向NULL，即可反转整个链表，且反转后的新链表的头结点是ptail。

空间复杂度分析：总共递归 n 层，系统栈的空间复杂度是 O(n)，所以总共需要额外O(n) 的空间。
时间复杂度分析：链表中每个节点只被遍历一次，所以时间复杂度是 O(n)。

5.表达式求值

难点在于 : 在表达式树中，如何判断某棵子树已经被遍历完。结合图示可以得出结论 : 当前读取到的运算符的优先级 <= 上一次读取到的运算符的优先级 ，则说明上一次读取到的运算符可以弹出相应的操作数进行计算了，该子树已经被遍历完。

还可以参考另一种思路 :

将中缀表达式转换为后缀表达式，然后求值.
基本策略: 碰到运算数 -> 输出
碰到运算符号 -> 存入堆栈，与后面碰到的运算符号比较优先级
左括号存入堆栈中时，优先级降到最低

具体步骤:

①运算数: 直接输出;
②左括号: 压入堆栈;
③右括号: 将栈顶的运算符弹出并输出，直到遇到左括号(出栈，不输出)
④运算符: 若优先级大于栈顶运算符时，则把它压栈;
若优先级小于等于栈顶运算符时，将栈顶运算符弹出并输出; 再比较新的栈顶运算符，直到该运算符大于栈顶运算符优先级为止, 然后将该运算符压栈;
⑤若各对象处理完毕，则把堆栈中存留的运算符一并输出.

8.LeetCode 73. 矩阵置零

原地算法 -> O(mn)

只需统计出矩阵中每一行或者每一列是否有 0 ，然后把含有 0 的行或者列都置成 0 即可。

①用两个变量记录第一行和第一列是否有0。
②遍历整个矩阵，用矩阵的第一行和第一列记录对应的行和列是否有0。
③把含有0的行和列都置成0。

时间复杂度分析：矩阵中每个元素只遍历常数次数，所以时间复杂度是 O(nm)。
空间复杂度分析：只用了两个额外的变量记录第一行和第一列是否含有 0，所以额外的空间复杂度是 (1)，满足原地算法的要求。

9.LeetCode 191.位1的个数

此题引入 lowbit(x) 函数，此函数的作用为返回参数x的二进制表示的最后一位1，

由图示推导并结合 -x 与 ~x + 1 的二进制表示完全相同之一特性，可以很方便的定义出 lowbit(x) 函数，而此题就是通过用 x 不断的减去 x 的二进制表示的最后一位1并同时进行计数，直到 x 为 0，得出位 1 的个数。
