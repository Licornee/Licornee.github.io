基于 GitHub Pages 应用 Hexo 在 macOS 环境下搭建个人主页

创建属于自己的GitHub Page

各大网站上有很多有关搭建博客的教程，作为一名入门选手，谨以此篇记录下自己亲自操作的流程和遇到的问题。

配置环境

Hexo 是基于 Node.js 的静态博客生成框架，所以在搭建博客之前需要先下载安装Node.js。

安装Node.js

通过点击小标题进入官网下载合适版本的pkj包进行安装，安装结束后打开终端，输入node -v，回车，正确显示版本号则可认定Node.js安装成功。

安装Git

通过点击小标题进入官网选择合适的方法安装Git（由于之前装过Homebrew，我采用 $ brew install git 的方法进行安装），安装结束后打开终端，输入git --version确认是否安装成功。

安装Hexo

在终端输入sudo npm install --unsafe-perm --verbose -g hexo命令，回车，安装结束后在终端输入hexo -v确认是否安装成功。

至此，Hexo博客基本环境配置完成。

搭建Hexo + GitHub静态博客，需要GitHub 账号，建立仓库用来托管我们的静态博客

建立托管博客的仓库

登录GitHub，点击New新建一个仓库

初始化

在合适的目录创建Blog文件夹作为博客根目录，在终端通过cd命令进入Blog文件夹，执行hexo init命令，Hexo 将在Blog文件夹自动新建所需要的文件。再执行npm install命令，完成本地博客文件夹初始化。

切换主题

打开位于博客根目录的_config.yml文件，找到并编辑其中theme: 字段，在其后添加主题名称，要注意的是 1. 主题名称要和/themes目录下的主题文件夹名称一致。2. 冒号后面要带有空格。

至此，在本地更换主题完成，在博客根目录执行命令hexo g，hexo s 预览，确认无误后执行hexo d，即可成功更换主题。
