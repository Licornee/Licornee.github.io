使用 Docker 安装 SQL Server

系统环境：macOS Big Sur 11.1

Docker版本：Docker 3.1.0

安装配置 Docker

什么是 Docker ?

Docker is a set of platform as a service (PaaS) products that use OS-level virtualization to deliver software in packages called containers.[6] Containers are isolated from one another and bundle their own software, libraries and configuration files; they can communicate with each other through well-defined channels.[7] All containers are run by a single operating system kernel and therefore use fewer resources than virtual machines.[8]

——Wikipedia

安装 Docker

点击标题中的超链接进入 Docker 官网下载 macOS 版本的软件并安装。

配置 Docker

由于 Docker 的镜像服务器在国外，为了更好的使用体验，需要对 Docker 进行配置，使用国内镜像源来加速下载。

进入 Docker 客户端，注册账号并登录后，点击 Dashboard 右上角的设置按钮，选择 Docker Engine选项，设置json为：

我使用的是网易的镜像源，国内还有很多其他的镜像源可以使用，例如：中国科技大学https://docker.mirrors.ustc.edu.cn ，七牛云 https://reg-mirror.qiniu.com 。

配置完成后点击右下角 Apply & Restart按钮，打开终端，输入 docker info命令即可查看相关信息，检查Registry Mirrors项是否已经更改为目标域名。

参数-e "SA_PASSWORD=<YourStrong@Passw0rd>"中<YourStrong@Passw0rd>(注意两端的尖括号)为SQL Server的登录密码，须指定为至少包含8个字符且符合SQL Server密码要求的强密码，在默认情况下，必须包含以下四种字符中的三种：大写字母、小写字母、十进制数字和符号。

参数--name sql1为容器指定一个自定义的名称，而不是使用随机生成的名称。如果 Docker 中运行多个容器，则无法重复使用相同的名称。(我选择将 sql1 替换为 sqlserver)

参数-h sql1用于显式设置容器主机名，如果不指定它，则默认为容器 ID ，该 ID 是随机生成的系统 GUID 。

将-h和--name设为相同的值是一种很好的方法，可以轻松地识别目标容器。

参数mcr.microsoft.com/mssql/server:2019-latest为容器镜像的名称。(一般默认即可)

从 DataGrip 连接至 SQL Server

打开 DataGrip ，按照 File -> Data Sources… -> + -> Microsots SQL Server 进入配置页面，输入刚才设置的用户名、端口、密码，点击 Test Connection 通过后就可以在 DataGrip 中使用 Docker 中的 SQL Server 了。注意：User 填入的并非是我们为容器指定的自定义名称， 按照上述命令创建的 SQL Server 容器的 User 为 SA。

至此，你已经可以成功的在 macOS上使用 SQL Server了。

解决 Datagrip 连接 SQL Server 后数据表中出现中文乱码的问题

从网上搜索了很多方案都称是 Datagrip 显示字体的问题，结果更改后发现并不能解决问题，于是尝试将表中列的数据类型由 char、varchar 切换为 nvarchar , 在执行 insert into 语句时在数据之前加上 N , 发现在查看表时中文就显示正常了。

那到底为什么会中文乱码呢？
究其原因，是因为在创建 SQLserver 数据库时未指定排序字符集，Databases则会使用instances的排序规则.
为了支持中文，需要设置成Chinese_PRC_CI_AS.
