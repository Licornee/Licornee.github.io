898.数字三角形（动态规划）

给定一个如下图所示的数字三角形，从顶部出发，在每一结点可以选择移动至其左下方的结点或移动至其右下方的结点，一直走到底层，要求找出一条路径，使路径上的数字的和最大。

输入格式

第一行包含整数 n，表示数字三角形的层数。

接下来 n 行，每行包含若干整数，其中第 i 行表示数字三角形第 i 层包含的整数。

输出格式

输出一个整数，表示最大的路径数字和。

数据范围

1 ≤ n ≤ 500，
−10000 ≤ 三角形中的整数 ≤ 10000

输入样例：

输出样例：

题解 O(n2)

首先考虑到暴力搜索的方法，共有 2n-1 条路径，而此题的数据范围 1 <= n <= 500，时间限制为 1s，必然会导致 Time Limit Exceeded，所以此路不通。

补充

由 数据范围反推算法复杂度以及算法内容 的规律，结合题目所给信息，应采用动态规划的方法。通过暴搜的方法，一次只能解决一个问题；通过动态规划的方法，一次能够解决一类问题，所以在时间复杂度上要小很多。

思路

首先，针对此题来讲，有两种思考的方向：①从上到下； ②从下到上。

如果选择从上到下的方向来解题，就要考虑对与路径上的每一个数，是从左上方读下来的还是从右上方读下来的，而这就会涉及到一些边界问题的特判，因为会有路径上的某些数字左上方的数不存在或者右上方的数不存在，要解决这样的边界问题，会导致代码冗长复杂，而选择从下到上的方向来解题，就不会出现这样的问题。

其次要考虑的就是如何用动态规划的思路解题，此处从集合的角度来进行分析。分析动态规划问题一般分两个步骤来考虑：①状态表示（成什么），即用哪个状态来表示一个集合；②状态（如何）计算。

在 状态表示 的步骤当中，我们要考虑的有两点：①该状态表示的是哪一个集合；②该状态表示的是集合的哪一个属性（一般有最大值、最小值和个数三类）。此题选择以 f(i, j)，即以三角形中点的坐标作为状态表示，该状态表示的是 从底向上读，读到点 (i, j) 这一点的所有路线 的集合，以此题所要求解的 最大值 作为属性。

在 状态计算 的步骤当中，一般要做的是 集合的划分 ，选定划分集合的依据将状态 f(i, j) 所表示的集合划分成若干个子集。此题中，选择路线中最后一个不相同的点作为划分的依据，将 f(i, j) 划分为两个子集，一个子集是从左下方向上读，即 (i + 1, j) -> (i, j) ;一个子集是从右下方向上读，即 (i + 1, j + 1) -> (i, j)。分别求出这两个子集的状态表示，选择其中的最大值加上 (i, j) 这一点的权值，即为 f(i, j) 。以此类推去递归地分析子集的值。

此题代码当中共有 n 2 个状态，以及与之对应的一个计算公式，所以时间复杂度为 O(n2).

Q：状态表示的方法是怎么想出来的？

A：从刷题经验中联想得出。对于此类路线问题，状态表示一般可以采用点的坐标来表示。

首先为了表示坐标点 (x, y) 上、下、左、右相邻的四个点的坐标，引入偏移量的概念，如图所示。

定义当前方向为变量 d，每当 “撞墙” 后更改 d 变量的值。“撞墙” 又分为两种情况，一种为出界，一种为重复碰到之前填过的格子。用变量 a 和 b 来表示从当前点沿当前方向 d 走到的下一个点的坐标，然后判断以 (a, b) 为坐标的点是否 ”撞墙“，若 ”撞墙“ 则更改方向 d 的值（顺时针旋转90°调整方向），并将坐标更新即可。

思路

用 left 变量指向待输出部分的最左侧，right 变量指向待输出部分的最右侧，top 变量指向待输出部分的最上侧，bottom 变量指向待输出部分的最下侧。

循环：

  1. 从左到右构造最上侧的一行，待出去部分的最上侧下移，然后 top+1。
  2. 从上到下构造最右侧的一列，待输出部分的最右侧左移，然后 right-1。
  3. 从右到左构造最下方的一行，待输出部分的最下侧上移，然后 bottom-1。
  4. 从下到上构造最右侧的一列，待输出部分的最左侧右移。然后 left+1。

直到：left > right && top > bottom
