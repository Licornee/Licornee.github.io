Have Fun with Numbers (20分)

Notice that the number 123456789 is a 9-digit number consisting exactly the numbers from 1 to 9, with no duplication. Double it we will obtain 246913578, which happens to be another 9-digit number consisting exactly the numbers from 1 to 9, only in a different permutation. Check to see the result if we double it again!

Now you are suppose to check if there are more numbers with this property. That is, double a given number with k digits, you are to tell if the resulting number consists of only a permutation of the digits in the original number.

Input Specification:
Each input contains one test case. Each case contains one positive integer with no more than 20 digits.

Output Specification:
For each test case, first print in a line “Yes” if doubling the input number gives a number that consists of only a permutation of the digits in the original number, or “No” if not. Then in the next line, print the doubled number.

思路

由于题目中提到

Each case contains one positive integer with no more than 20 digits.

所以使用int、long、long long类型都无法存储这个数字，应选用char类型的数组。用num数组来存储读入的数字，用dnum数组存储double之后的数字，用book数组来记录num数组中每个数字出现的次数，然后用它来跟double之后的数字进行操作判断是否满足题意。

声明carry变量存储进位。将数字double之后如果有进位，进位一定进1，同时考虑到如果数字的第一个数字在double之后或者被进位+1之后会产生进位导致整个数字的位数加1，所以留出dnum数组的第一位(即dnum[0])，若位数增加，则先打印出第一位1，若位数不变，则从dnum数组的第二位开始(即dnum[1])向数组结尾打印。

通过声明flag变量

to tell if the resulting number consists of only a permutation of the digits in the original number.

如果flag == 1, 则Yes; 如果flag == 0, 则No.

此题的关键是double之后每个位上数字进位的处理，此方法的巧妙之处在于通过使用两个数组，将dnum[0]预留出来，根据数字的第一位数字是否产生进位来分情况输出。

①从题目所给的输入、输出样例中可以看到输入15，输出1307674368000，位数为13位，而题目中提到N的最大值不超过1000，所以当N的值达到1000左右时，所要输出的数值大小远超过int、long、long long类型所能表示的数值范围。

解决方法：通过一个大的数组来存储所要输出的数值各个位上的数字。

补充：由上述解决方法的思路，如何确定数组的大小便成了所要考虑的问题，由此引入斯特林公式（Stirling’s approximation）来得到阶乘的位数，从而确定数组的大小。

斯特林公式是一条用来取n的阶乘的近似值的数学公式。一般来说，阶乘的计算复杂度为线性。当要为某些极大大的n求阶乘时，常见的方法复杂度不可接受。斯特林公式能够将求解阶乘的复杂度降低到对数级。而且，即使在n很小的时候，斯特林公式的取值已经十分准确。

公式形式：

由数学知识可知:设一个数为n，则此数的位数 r = lg(n) + 1
而求一个数n的阶乘的位数 r，则r = lg(n!) + 1
